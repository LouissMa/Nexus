# 🗺️ NEXUS Development Roadmap

This document outlines the strategic phases for NEXUS. We are currently in **Phase 1**.

## Phase 1: The Memory & Proactive MVP (Current)
*Objective: Prove the core loop of memory extraction and proactive triggering.*
- [ ] Establish basic project structure and documentation.
- [ ] Implement CLI (Command Line Interface) or basic Bot for initial testing.
- [ ] Build basic Vector Database connection (e.g., Pinecone/ChromaDB) for Long-Term Memory.
- [ ] Implement a Cron-job system to trigger proactive messages based on user inactivity.

## Phase 2: Context & Habit Tracking
*Objective: Make the AI smarter about daily routines.*
- [ ] Implement structured habit tracking in the database.
- [ ] Develop dynamic prompt engineering to adjust the AI's "tone" based on user progress.
- [ ] Add a basic dashboard for users to visualize their goals.

## Phase 3: The "Life OS" Integrations
*Objective: Connect NEXUS to the user's existing digital life.*
- [ ] Calendar integration (AI can proactively suggest scheduling blocks).
- [ ] Email triage (AI summarizes daily incoming emails).
- [ ] Task management automation.

## Phase 4: The Digital Twin
*Objective: High emotional intelligence and autonomous task execution.*
- [ ] Deep emotional analysis and adaptive persona generation.
- [ ] Autonomous agent capabilities (executing routine tasks with permission).

# 🗺️ NEXUS 开发路线图

本文档概述了 NEXUS 的战略发展阶段。我们目前正处于 **第一阶段 (Phase 1)**。

## 阶段一：记忆与主动触发 MVP (当前阶段)
*目标：跑通“记忆提取”和“主动触发”的核心闭环。*
- [ ] 建立基础项目结构和专业文档。
- [ ] 实现命令行 (CLI) 或基础 Bot 接口用于早期测试。
- [ ] 接入向量数据库（如 Pinecone/ChromaDB），实现长期记忆 (Long-Term Memory)。
- [ ] 实现定时任务 (Cron-job) 系统，当用户未达成目标时触发 AI 主动推送。

## 阶段二：上下文与习惯追踪
*目标：让 AI 更智能地理解用户的日常作息。*
- [ ] 在数据库中实现结构化的习惯打卡追踪。
- [ ] 开发动态提示词工程 (Prompt Engineering)，根据用户的进度调整 AI 的“语气”。
- [ ] 增加基础的数据看板，让用户直观看到自己的目标进度。

## 阶段三：“个人系统”工具整合
*目标：将 NEXUS 接入用户现有的数字生活。*
- [ ] 日历整合（AI 可主动建议并安排日程）。
- [ ] 邮件处理（AI 每日总结重要邮件）。
- [ ] 任务管理自动化。

## 阶段四：终极数字分身
*目标：具备高情商并能自主执行任务。*
- [ ] 深度情感分析与自适应人设生成。
- [ ] 自动化 Agent 能力（在授权下自动执行日常琐事）。
